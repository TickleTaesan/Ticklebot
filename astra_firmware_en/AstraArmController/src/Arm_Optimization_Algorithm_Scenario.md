# 팔 최적화 알고리즘 시나리오 (AstraArmController)

1. **초기화 단계**
   - 펌웨어가 부팅되면 시리얼 통신, 설정값, 모터 제어, 토크 상태를 초기화한다.
   - ROS2 노드(arm_node.py)와의 연결이 준비된다.

2. **명령 수신**
   - ROS2 노드에서 새로운 관절 위치 명령(Position Command)이 시리얼로 전송된다.

3. **(선택) EIShaper 필터 적용**
   - 진동 억제를 위해 EIShaper 알고리즘이 활성화되어 있으면, 목표 위치 명령에 필터가 적용된다.

4. **트라페zoidal 트래젝토리 플래닝**
   - 각 관절별로 트라페zoidal(사다리꼴) 속도 프로파일을 생성하여, 가속-정속-감속의 부드러운 경로를 계획한다.

5. **트래젝토리 진행 및 세트포인트 갱신**
   - 제어 주기마다 현재 시간에 맞는 위치/속도 세트포인트를 계산한다.

6. **PID 제어 (듀얼 게인 포함)**
   - 각 관절별로 듀얼 게인 P, I, D, 안티윈드업, 스틱션 보상 등 고급 PID 제어를 수행한다.
   - 오차가 작을 때는 낮은 게인, 클 때는 높은 게인을 적용해 빠르면서도 정밀한 제어를 달성한다.

7. **TTL 신호 송출**
   - 계산된 제어 출력을 각 서보 모터에 TTL 신호로 전송한다.

8. **엔코더 피드백 읽기**
   - 각 관절의 실제 위치를 엔코더로 읽어온다.

9. **피드백 송신**
   - 현재 관절 상태(위치, 속도 등)를 ROS2 노드로 시리얼을 통해 전송한다.

10. **루프 반복**
    - 새로운 명령이 오면 3단계부터 반복, 없으면 5단계부터 계속 진행한다.

---

이 시나리오는 AstraArmController 펌웨어의 실시간 최적화 제어 과정을 단계별로 설명합니다. 각 단계는 실제 코드 구조 및 동작 흐름에 기반합니다.
